Methodology

VERDICT Framework v0.3.1 evaluates AI agent platforms across seven dimensions on an 85-point scale using public data only. This page documents the scoring system, evaluation layers, rules, and vendor correction process.

Scoring Dimensions

Each platform is scored across seven dimensions. The acronym V-E-R-D-I-C-T maps to the evaluation areas. Layer 0 (public documentation only) has a maximum of 85 points because the Effectiveness dimension requires behavioral testing (Layer 1).

V — Verifiability20 pts
Can the platform's security claims be independently verified? Covers developer identity, source code disclosure, version management, dependency transparency, independent certifications, and documentation quality.
E — Effectiveness15 pts · Layer 1
Does the platform perform securely under real-world conditions? Requires behavioral testing (30 runs across 4 difficulty levels). Excluded from Layer 0 evaluations. Planned for Q2 2026.
R — Resilience20 pts
How does the platform handle vulnerabilities? Covers CVE count (trailing 12 months), maximum CVSS severity, patch response speed, recurring structural issues, and supply chain compromise history. A CVSS 9.0+ penalty applies.
D — Data Conduct15 pts
How is user data handled? Covers GDPR compliance, data minimization (telemetry posture), AI model training policy, sub-processor transparency, and data retention disclosure.
I — Identity & Control10 pts
Can humans maintain control over the AI agent? Covers emergency stop mechanisms, human-in-the-loop design, and permission delegation chain transparency (auth, RBAC, audit logging).
C — Containment10 pts
Are agent actions bounded? Covers sandbox design philosophy, least privilege enforcement, and tenant isolation. The weakest average dimension in the index (4.9/10 across Q1 platforms).
T — Transparency10 pts
Does the platform openly communicate its security posture? Covers CVE publication posture, incident disclosure speed, security policy publication, AI safety framework adoption, and AI system identity disclosure.

Evaluation Layers

Layer 0
Active · 85 pts max
Public documentation only. CVE databases (NVD, MITRE), CISA KEV catalog, GitHub Security Advisories, official documentation, Terms of Service, privacy policies, and compliance certifications. No vendor accounts or internal access. Zero cost.
Layer 1
Planned · Q2 2026
Behavioral testing on free tiers or self-hosted deployments. 30 runs across 4 difficulty levels per platform. Adds the Effectiveness (E) dimension, bringing the maximum to 100 points.
Layer C
Planned · Continuous
Live CVE monitoring and compliance update tracking. Triggers re-evaluation when new vulnerabilities or certification changes are detected.

Scoring Rules

Silence = Data PointIf a platform does not publicly document a security control, it is scored as absent (zero). The absence of documentation is itself informative.
No Intent AttributionEvaluations never attribute motives to vendors. "No vulnerability disclosure program was identified" — not "the vendor chose not to publish one."
Cite Every ClaimEvery factual statement references a public source: CVE numbers, official documentation URLs, compliance certification pages, or advisory databases.
Include Positive FindingsEvery evaluation records strengths alongside concerns. VERDICT is a witness, not a prosecution.
Risk Factors, Not Pass/FailScores record risk factors by use case (internal testing, credential workflows, multi-tenant, regulated data). No platform is "recommended" or "not recommended."
CISA KEV FlagPlatforms with entries in the CISA Known Exploited Vulnerabilities catalog are flagged. This is a factual marker, not a scoring penalty beyond the CVE's existing impact on the R dimension.
TiebreakingEqual scores are sorted alphabetically by platform name.
Bias DisclosureEvery evaluation includes a bias disclosure statement identifying VERDICT's use of Claude (Anthropic) as tooling and any known commercial relationships between Anthropic and the evaluated vendor.

Evaluation Process

1
Research. Public data is collected from CVE databases, official documentation, GitHub repositories, compliance certification pages, and privacy policies. No vendor accounts are created.
2
Score. Each dimension is scored against documented criteria. Dimension totals are summed to produce the overall score. The evaluation undergoes a 3-pass quality review (factual accuracy, legal risk, audience perception).
3
Publish. The evaluation page goes live on getverdict.fyi with dimension breakdowns, radar charts, incident timelines, and bias disclosures.
4
Notify. The vendor receives a post-publication email summarizing key findings and inviting factual corrections with supporting public documentation. No response is required.
5
Correct. If a vendor provides verifiable factual corrections, the score is updated promptly. The correction is documented in the evaluation record. To date, one score correction has been applied through this process (Dust: 55→57, Transparency dimension updated after vendor confirmed the existence of a VDP, SECURITY.md, and security.txt).

What VERDICT Is Not

VERDICT is not a certification authority. A VERDICT score does not constitute a security certification, endorsement, or guarantee. Scores reflect the state of publicly available documentation at the time of evaluation. Platforms may have security controls that are not publicly documented and therefore not captured by Layer 0.

VERDICT does not conduct penetration testing, access vendor infrastructure, or use vendor accounts at Layer 0. Behavioral testing (Layer 1) is planned for Q2 2026 and will use only free tiers or self-hosted deployments.
